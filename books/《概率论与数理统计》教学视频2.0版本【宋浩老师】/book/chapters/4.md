```python
import sympy as sp
from sympy.stats import FiniteRV, E

X = FiniteRV("X", {0:0.1,1:0.2,2:0.3,3:0.4})
# 左边 X（等号前面）：给代码、程序用，用来做计算
# 引号里 "X"（函数第一个参数）：纯给人看，打印输出时显示在公式里
# 0,1,2,3 不是随机变量，只是随机变量 X 能取到的取值（支撑集 \(S_X\)）
# 等号左边的 X 这个对象 = 完整随机变量，它内部同时存了三样东西：
# 能取的值：0,1,2,3
# 每个值对应的概率（PMF）
# 标识名 "X"
# 只有这个打包好的对象，才能用 E(X)、P(X>2) 算期望、概率；
# # 为什么 FiniteRV 封装了 pmf，还叫随机变量：
# 1. 数学逻辑：随机变量先存在，PMF 是它的派生产物
# 2. SymPy 工具逻辑：反过来，用 PMF 完整代表随机变量
# 3. 正确的关系描述
# 数学上：随机变量是本体，PMF 是本体生成的附属函数；
# 代码里：PMF 是唯一标识本体统计行为的完整特征，库直接拿这套特征打包，做成一个能直接运算的代理对象，把这个代理命名为随机变量；
# PMF 不是随机变量的“某一小块零件”，而是能完全替代它做所有统计计算的完整特征集，被封装在随机变量对象内部作为核心数据。

print(X)

E(X)
```

### FiniteRV 打包后的产物我们可以直接就认为是随机变量，当随机变量用

#### 结论完全正确

在**代码计算、做题求期望/概率**这个场景里，你可以 100% 把 `FiniteRV` 打包出来的 `X` 当成数学里的随机变量直接使用，不用纠结缺失原始样本空间的理论细节。

##### 1. 为什么能直接当随机变量用？

所有你会用到的概率论计算公式，只依赖 PMF，完全不依赖底层样本空间 $\Omega$：

- 求变换期望：$E[g(X)]=\sum g(x)f(x)$
- 求事件概率：$P(a<X<b)=\sum\limits_{a<x<b}f(x)$
- 方差、矩、分布变换，全部只需要「取值集合+PMF」

打包后的对象内部刚好存了这两样核心数据，同时内置了适配随机变量的运算规则：

```python
from sympy import S
from sympy.stats import P

X = FiniteRV("X", {-1:S("1/3"),0:S("1/3"),1:S("1/3")})
#### 完全对标数学随机变量的用法
print(E(X))    # 一阶期望
print(E(X**2)) # 函数变换后的期望
print(P(X > 0.1))# 随机变量满足不等式的概率
```

只要是数学课上能对随机变量做的数值计算，这个打包对象全能实现，行为完全对齐。

##### 2. 唯一需要区分的边界（只在测度论定义场景有用）

只有当你在严格公理化、测度论层面讨论「随机变量是 $\Omega\to\mathbb{R}$ 的映射」时，它才不算完整定义的随机变量（缺少原始样本空间那层映射）；
但只要是**计算、做题、数值分析**，这个边界完全可以忽略，直接把 `FiniteRV` 的返回物视作随机变量即可。

##### 3. 通俗总结

- 理论课：完整随机变量 = 样本空间映射 + 诱导出来的 PMF；
- 代码实操：封装好的 `X` = 等效随机变量，仅凭 PMF 就能完成全部计算，直接拿来用没有任何问题。

### [3]

#### 一、题目说明

连续随机变量 \(X\) 的概率密度：
\[
f(x)=
\begin{cases}
2x, & 0<x<1 \\
0, & \text{其他}
\end{cases}
\]
求数学期望 \(E(X)=\displaystyle\int\_{-\infty}^{+\infty}x\,f(x)\,dx\)。

##### 解析计算

\[
E(X)=\int_0^1 x\cdot 2x\,dx
=\int_0^1 2x^2 dx
= \left.\frac{2}{3}x^3\right|\_0^1
=\frac{2}{3}
\]

#### 二、代码实现（3 种方式）

##### 1. Python 符号计算（SymPy，精确解析解）

```python
import sympy as sp
sp.init_printing()

x = sp.Symbol("x")
# 先限定 0<x<1，剩下所有情况为 0
f = sp.Piecewise( # Piecewise 是 SymPy 库专门用来定义分段函数 的类，直译：分段、分块
    (2*x, (x > 0) & (x < 1)),  # 第一优先级：0<x<1
    (0, True)                   # 其余全部情况：x≤0 或 x≥1
)

EX = sp.integrate(x * f, (x, -sp.oo, sp.oo))
print("E(X) =", EX)
EX
```

输出：`E(X) = 2/3`

##### 2. Python 数值积分（Scipy，数值近似）

```python
from scipy import integrate

def pdf(x):
    if 0 < x < 1:
        return 2 * x
    else:
        return 0

#### 积分 x*pdf(x) 从 0 到 1
EX, err = integrate.quad(lambda t: t * pdf(t), 0, 1)
print(f"E(X) ≈ {EX}")
```

输出：`E(X) ≈ 0.6666666666666666`

##### 3. 蒙特卡洛模拟（随机采样验证）

```python
import numpy as np

#### 逆变换采样：CDF F(x)=x² → 逆函数 x=√u, u~Uniform(0,1)
np.random.seed(42)
N = 1000000
u = np.random.rand(N)
samples = np.sqrt(u)
EX_mc = samples.mean()
print(f"蒙特卡洛估计 E(X) ≈ {EX_mc}")
```

输出会接近 `0.666...`，样本量越大越接近 \(\dfrac{2}{3}\)。

##### 4. R 语言版本

```python
from scipy import integrate

# 纯数值函数，可直接被 quad 调用
def f_num(x):
    return 2 * x if 0 < x < 1 else 0

# 计算 E(X) = ∫ t*f_num(t) dt
EX, err = integrate.quad(lambda t: t * f_num(t), 0, 1)
print(f"数值积分 E(X) = {EX}")
```

#### 三、补充说明

1. 数学期望本质是**加权积分**，权重为概率密度 \(f(x)\)；
2. 符号计算能直接得到精确分数 \(\dfrac{2}{3}\)，数值积分/蒙特卡洛是近似验证；
3. 该分布是 Beta 分布 \(\text{Beta}(2,1)\)，理论期望 \(\dfrac{\alpha}{\alpha+\beta}=\dfrac{2}{3}\)，和计算结果一致。

## 期望有没有函数，得自己构造吗？

分两层说：**SymPy 没有单独叫 `expectation()` 顶层万能函数，但有专用工具，不用纯手动拼积分/求和**

### 一、先纠正：不用完全自己从零构造积分/求和

SymPy 内置 `sympy.stats.E()`，专门计算随机变量数学期望，是官方期望函数，不用手动写 `integrate` / `summation`。

#### 示例 离散分布

```python
from sympy.stats import DiscreteRV, E
from sympy import symbols, Piecewise
import sympy as sp

x = symbols("x", integer=True)
## 构造 PMF
pmf = Piecewise((0.6, sp.Eq(x, 1)), (0.4, sp.Eq(x, 2)), (0, True))
## 定义离散随机变量 X
X = DiscreteRV(x, pmf)
# Discrete = 离散的
# RV = Random Variable（随机变量）
# 全称：Discrete Random Variable，离散随机变量
# DiscreteRV 做的事：把「符号自变量 + PMF 表达式 + 取值支撑集」三者打包，生成并返回一个离散随机变量对象；
# 这一步做了完整封装，内部绑定三样东西：
# 自变量 x；
# 你提供的函数 f，并规定这个 f 就是该随机变量的 PMF；
# 变量取值定义域 set。
# 封装完成后得到一个随机变量对象 X，而不是单纯的函数。
print(X)

## 直接求期望，不用手写求和
EX = E(X)
print(EX)

sp.init_printing()
EX
```

#### 连续分布

```python
from sympy.stats import ContinuousRV, E
from sympy import symbols, Piecewise

t = symbols("t")
pdf = Piecewise((2*t, sp.And(0 < t, t < 1)), (0, True))
T = ContinuousRV(t, pdf)
ET = E(T)
print(ET)
```

### 二、两种使用路线对比

#### 路线 1：用 `sympy.stats.E()`（推荐，标准期望接口）

优点：

1. 自动区分离散/连续，**内部自动调用 summation / integrate**；
2. 支持复合函数期望：$E(g(X)), E(XY), Var(X)$ 配套工具；
3. 自动处理无穷区间、支撑集，不用手动写 `-oo, oo`。

缺点：需要先按 SymPy 规范定义 `DiscreteRV` / `ContinuousRV` 随机变量对象。

#### 路线 2：自己手动写积分/求和（你说的“自己构造”）

summation：符号求和，用于离散期望 \(E[X]=\sum x\cdot pmf(x)\)
integrate：符号积分，用于连续期望 \(E[X]=\int t\cdot pdf(t) dt\)
oo：无穷大符号 \(\infty\)

```python
from sympy import summation, integrate, oo

## 离散
summation(x * pmf, (x, -oo, oo))
## 连续
integrate(t * pdf, (t, -oo, oo))
```

适用场景：

- 只是临时算一步，不想引入 `sympy.stats` 整套随机变量体系；
- 自定义非常复杂的分段密度，不想封装 RV 对象；
  本质是手动实现期望的定义式。

### 三、关键结论

1. **存在专用期望函数 `E()`**，不是完全没有；
2. 如果你不想用 `sympy.stats` 模块，只靠 Piecewise 密度/质量函数，就必须**自己写求和/积分**，等价于手动实现期望公式；
3. 两者数学等价：
   $$E(X) = \int_{-\infty}^{\infty} x f(x)dx = \sum_x x p(x)$$
   `E(X)` 只是把这段重复代码封装好了，底层依然是积分/求和。

### 补充：精算常用场景建议

- 批量算寿险、年金现金流期望、复合损失分布：优先 `sympy.stats.E()`，代码简洁；
- 临时验证分段 PMF/PDF、手推公式验算：直接手动 integrate / summation 更直观。
