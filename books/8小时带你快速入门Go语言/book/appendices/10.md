# 不只是 struct，所有我们自定义的类型都可以挂载方法？

对。**不只是 `struct`，只要是你在当前包中定义的“具名类型”，通常都可以挂方法。**

### 基础类型的自定义类型

```go
type Celsius float64

func (c Celsius) Fahrenheit() float64 {
	return float64(c)*9/5 + 32
}

temperature := Celsius(30)
fmt.Println(temperature.Fahrenheit())
```

### 切片类型

```go
type Numbers []int

func (nums Numbers) Sum() int {
	total := 0
	for _, n := range nums {
		total += n
	}
	return total
}

nums := Numbers{1, 2, 3}
fmt.Println(nums.Sum()) // 6
```

### map 类型

```go
type StringSet map[string]bool

func (s StringSet) Add(value string) {
	s[value] = true
}

set := StringSet{}
set.Add("Go")
```

### 函数类型

```go
type Operation func(int, int) int

func (op Operation) Calculate(a, b int) int {
	return op(a, b)
}

add := Operation(func(a, b int) int {
	return a + b
})

fmt.Println(add.Calculate(2, 3)) // 5
```

数组、通道等具名类型也可以挂方法。

不过不能直接给未命名类型挂方法：

```go
func (nums []int) Sum() int { // 错误
	return 0
}
```

必须先定义一个名字：

```go
type Numbers []int

func (nums Numbers) Sum() int {
	return 0
}
```

也不能给其他包定义的类型直接添加方法：

```go
func (t time.Time) Hello() {} // 错误
```

需要包装或重新定义：

```go
type MyTime time.Time

func (t MyTime) Hello() {
	fmt.Println("hello")
}
```

因此核心规则是：

> 方法可以挂在当前包自己定义的具名类型上，`struct` 只是最常见的一种。

# 什么是具名类型？

**具名类型**就是：这个类型有一个你定义的名字，可以用这个名字反复引用。

例如：

```go
type Celsius float64
```

这里：

- `float64` 是底层类型
- `Celsius` 是你新定义的具名类型

所以可以给它挂方法：

```go
func (c Celsius) Fahrenheit() float64 {
	return float64(c)*9/5 + 32
}
```

## 结构体也一样

```go
type User struct {
	Name string
}
```

`User` 是具名类型，底层类型是：

```go
struct {
	Name string
}
```

因此可以挂方法：

```go
func (u User) Greeting() string {
	return "Hello, " + u.Name
}
```

而这个结构体没有类型名：

```go
user := struct {
	Name string
}{
	Name: "Tom",
}
```

这里的：

```go
struct {
	Name string
}
```

是**匿名类型／未命名类型**，不能直接给它声明方法。

## 切片的例子

```go
[]int
```

这是未命名的切片类型。

给它起一个新类型名：

```go
type Numbers []int
```

现在 `Numbers` 就是具名类型，可以挂方法：

```go
func (nums Numbers) Sum() int {
	total := 0
	for _, n := range nums {
		total += n
	}
	return total
}
```

## `type` 别名要注意

下面不是创建新类型：

```go
type MyInt = int
```

这是**类型别名**，`MyInt` 实际上仍然就是 `int`，不能靠它给 `int` 添加方法。

而：

```go
type MyInt int
```

才是创建一个新的具名类型，可以挂方法：

```go
func (n MyInt) Double() MyInt {
	return n * 2
}
```

可以简单记：

```go
type User struct{} // 新的具名类型
type Numbers []int // 新的具名类型
type MyInt int     // 新的具名类型
type MyInt = int   // 只是别名，不是新类型

[]int    // 未命名类型
struct{} // 未命名类型
```

所以“给类型挂方法”的核心条件之一就是：

> 这个类型必须是当前包中定义的新类型，而不能只是临时写出来的匿名类型或其他类型的别名。

# 类型是用大驼峰是吗？

一般是。Go 的类型名通常使用驼峰命名：

```go
type User struct{}
type UserService struct{}
type HTTPClient struct{}
type OrderID int
```

首字母大写时，类型可以被其他包使用：

```go
type User struct{}
```

其他包可以写：

```go
user.User{}
```

首字母小写时，类型只在当前包内部可见：

```go
type userCache struct{}
type internalConfig struct{}
```

所以不是所有类型都必须大驼峰，而是：

```text
需要跨包导出 → 大驼峰
只在包内使用 → 小驼峰
```

缩写通常保持完整大写：

```go
type UserID int
type APIClient struct{}
type HTTPServer struct{}
```

一般不写：

```go
type UserId int
type ApiClient struct{}
type HttpServer struct{}
```

另外，Go 官方更常称这种风格为 `MixedCaps`，而不是强调“大驼峰”这个术语。

# 那基本所有的都是大驼峰啊

你会觉得“基本都是大驼峰”，是因为我们前面举的很多例子都是**要跨包使用的导出名字**。

Go 真正的规则不是“都用大驼峰”，而是：

> **首字母大写表示导出，首字母小写表示只在当前包内使用。**

例如：

```go
type User struct { // 导出的类型
	ID       int    // 导出的字段
	username string // 包内字段
}

func NewUser() *User { // 导出的函数
	return &User{}
}

func validateUser() {} // 包内函数

func (u *User) Save()     {} // 导出方法
func (u *User) validate() {} // 包内方法
```

实际项目里，小驼峰的名字非常多：

```go
user := NewUser()
userID := 10
totalCount := 20
err := saveUser(user)
```

包括：

- 局部变量：`userName`
- 参数：`userID`
- 接收者：`u`
- 包内函数：`findUser`
- 包内类型：`userCache`
- 私有字段：`passwordHash`
- 包名：`user`、`http`、`config`

而大驼峰主要出现在公共 API：

```go
User
UserService
NewUser
FindByID
Save
HTTPClient
```

所以更准确地说：

```text
公共、跨包使用的名字 → 大驼峰
包内、局部使用的名字 → 小驼峰
```

Go 项目里通常是**小驼峰更多，大驼峰更显眼**。
