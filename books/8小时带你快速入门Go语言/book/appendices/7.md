## 字符串不能用单引号

Go 里的字符串可以用两种写法：

```go
s1 := "hello"
s2 := `hello`
```

也就是：

- 双引号 `"..."`：普通字符串，支持 `\n`、`\t`、`\"` 等转义
- 反引号 `` `...` ``：原始字符串，不处理转义，还可以跨行

单引号 `'...'` 不是字符串，而是 `rune`：

```go
c := '你' // rune
```

所以准确说：**Go 字符串不只是双引号，也可以用反引号，但不能用单引号。**

## 传入的参数一样时，类型声明写一个就可以吗？

可以。Go 里连续参数类型相同时，只写最后一个类型即可：

```go
func area(l, w float64) float64 {
	return l * w
}
```

等价于：

```go
func area(l float64, w float64) float64 {
	return l * w
}
```

更多参数也一样：

```go
func add(a, b, c int) int {
	return a + b + c
}
```

类型不同时就要分别写：

```go
func greet(name string, age int) {
}
```

返回值也能这样合并：

```go
func calc() (x, y int) {
	return 1, 2
}
```

这里 `x`、`y` 都是 `int`。

## `[]int{1, 2, 3}`是什么

`[]int{1, 2, 3}` 是一个 **`int` 类型的切片字面量**。

可以拆开看：

```go
[]int{1, 2, 3}
```

- `[]int`：类型，表示 `int` 切片
- `{1, 2, 3}`：切片里的初始元素

通常这样赋值：

```go
nums := []int{1, 2, 3}
```

这时：

```go
fmt.Printf("%T\n", nums)
```

输出：

```text
[]int
```

它和数组不一样：

```go
a := []int{1, 2, 3}  // 切片
b := [3]int{1, 2, 3} // 长度为 3 的数组
```

区别在于：

```go
[]int
```

中括号里没有长度，所以是切片；

```go
[3]int
```

中括号里有长度，所以是数组。

还可以创建空切片：

```go
nums := []int{}
```

然后追加元素：

```go
nums = append(nums, 1)
nums = append(nums, 2)
```

所以你可以把 `[]int{1, 2, 3}` 理解成：

> 创建一个装有 `1、2、3` 的 `int` 切片。

## 切片和数组的区别就是有没有数字，是不是定长？

对，**最直观的区别就是中括号里有没有长度数字**：

```go
a := [3]int{1, 2, 3} // 数组
b := []int{1, 2, 3}  // 切片
```

数组是定长的，而且长度属于类型：

```go
var a [3]int
var b [4]int
```

`[3]int` 和 `[4]int` 是两种不同类型，数组创建后长度不能改变。

切片长度可以变化：

```go
s := []int{1, 2, 3}
s = append(s, 4)
```

不过更准确地说，切片本身不是“无限长数组”，而是一个指向底层数组的描述信息，大致包含：

```text
底层数组的位置 + 当前长度 + 容量
```

还有一个重要区别：数组赋值会复制整个数组：

```go
a := [3]int{1, 2, 3}
b := a

b[0] = 100

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [100 2 3]
```

切片赋值通常共享同一个底层数组：

```go
a := []int{1, 2, 3}
b := a

b[0] = 100

fmt.Println(a) // [100 2 3]
fmt.Println(b) // [100 2 3]
```

所以可以先这样记：

> `[3]int` 是固定长度数组，`[]int` 是可变长度切片。日常 Go 代码里切片用得更多。

## 迭代只能用 range 吗？

不是。Go 只有一个循环关键字 `for`，`range` 只是其中一种遍历方式。

#### 使用 `range`

```go
nums := []int{10, 20, 30}

for index, value := range nums {
	fmt.Println(index, value)
}
```

只要值：

```go
for _, value := range nums {
	fmt.Println(value)
}
```

#### 使用普通下标循环

```go
for i := 0; i < len(nums); i++ {
	fmt.Println(nums[i])
}
```

它适合需要控制步长、倒序或修改下标的时候：

```go
for i := len(nums) - 1; i >= 0; i-- {
	fmt.Println(nums[i])
}
```

#### 类似 `while`

Go 没有 `while`，直接写：

```go
i := 0

for i < len(nums) {
	fmt.Println(nums[i])
	i++
}
```

#### 无限循环

```go
for {
	fmt.Println("一直执行")
}
```

因此遍历切片时：

```go
for _, value := range nums
```

最方便，但并不是只能使用 `range`。普通的 `for i := 0; ...` 同样常用。

## `func add(nums ...int) int {`可变参数

这是 Go 的**可变参数**：

```go
func add(numberList ...int) {
}
```

`...int` 表示：

> 调用 `add` 时，可以传入任意数量的 `int`，包括 0 个。

例如：

```go
add()
add(1)
add(1, 2, 3)
add(10, 20, 30, 40)
```

在函数内部，`numberList` 会被当成一个 `[]int` 切片：

```go
func add(numberList ...int) {
	fmt.Printf("%T\n", numberList) // []int

	for _, n := range numberList {
		fmt.Println(n)
	}
}
```

可以用它实现求和：

```go
func add(numberList ...int) int {
	total := 0

	for _, n := range numberList {
		total += n
	}

	return total
}

result := add(1, 2, 3)
fmt.Println(result) // 6
```

它和普通切片参数的区别是调用方式不同：

```go
func add1(nums []int)  {}
func add2(nums ...int) {}
```

调用：

```go
add1([]int{1, 2, 3})
add2(1, 2, 3)
```

已有切片也可以传给可变参数，但要在后面加 `...`：

```go
nums := []int{1, 2, 3}

add2(nums...)
```

这里两个 `...` 的位置含义不同：

```go
func add(nums ...int) // 声明：接收任意多个 int

add(nums...) // 调用：把切片展开成多个参数
```

另外，可变参数必须放在参数列表最后：

```go
func printAll(prefix string, nums ...int) {}
```

不能写成：

```go
func printAll(nums ...int, prefix string) {} // 错误
```

## 具名返回值

### 这个具名有什么用，只是语义上有用吗？

不只是语义上有用。**具名返回值会直接成为函数体里的局部变量。**

例如：

```go
func divide(a, b int) (result int, ok bool) {
	if b == 0 {
		return 0, false
	}

	result = a / b
	ok = true
	return
}
```

这里的：

```go
result int
ok bool
```

在进入函数时就已经被声明了，并且具有零值：

```go
result == 0
ok == false
```

所以可以直接赋值：

```go
result = a / b
ok = true
```

最后只写：

```go
return
```

这种写法叫 **裸返回**，它会返回当前的 `result` 和 `ok`。

---

不具名时则要自己声明并明确返回：

```go
func divide(a, b int) (int, bool) {
	if b == 0 {
		return 0, false
	}

	result := a / b
	return result, true
}
```

具名返回值还有一个比较特殊的用途：`defer` 可以访问甚至修改它。

```go
func calculate() (result int) {
	defer func() {
		result++
	}()

	result = 10
	return
}
```

最终返回的是：

```text
11
```

因为执行顺序是：

```text
result = 10
准备 return
执行 defer：result++
真正返回 11
```

不过这种修改返回值的技巧容易让代码变隐式，通常不建议滥用。

具名返回值也有文档意义：

```go
func lookup(key string) (value string, found bool)
```

比：

```go
func lookup(key string) (string, bool)
```

更容易看出两个返回值分别代表什么。

所以它主要有三个作用：

1. 给返回值增加语义。
2. 返回值可直接作为函数内变量使用。
3. 允许裸 `return`，也能被 `defer` 访问。

实际写代码时，简单函数通常不必具名：

```go
func add(a, b int) int {
	return a + b
}
```

返回值较多、含义不明显时，具名会更有价值：

```go
func parse(input string) (value int, valid bool, err error)
```

### 噢，函数内可以直接用，函数外用的话还得声明是吗？

对。具名返回值的名字只在函数内部有效，函数外不能直接使用那个名字。

```go
func lookup() (val string, ok bool) {
	val = "hello"
	ok = true
	return
}
```

调用者仍然要自己接收：

```go
value, found := lookup()
```

这里外面的变量可以随便起名，不必叫 `val` 和 `ok`：

```go
name, exists := lookup()
```

具名返回值不会把变量名传到函数外，只会把两个值传出来。

已有变量时，用 `=`：

```go
var value string
var found bool

value, found = lookup()
```

不需要某个返回值时，可以用 `_` 丢掉：

```go
value, _ := lookup()
```

所以作用域大致是：

```go
func lookup() (val string, ok bool) {
	// val、ok 在这里可以直接使用
}

// 这里没有 val、ok
// 必须重新声明变量接收返回值
value, found := lookup()
```

### 如果声明了具名返回，函数内部必须使用这个变量？

不必须。声明了具名返回值后，既可以使用它们，也可以完全不用，直接明确返回值：

```go
func lookup() (val string, ok bool) {
	return "hello", true
}
```

虽然声明了 `val` 和 `ok`，函数里并没有给它们赋值，这完全合法。

也可以返回其他局部变量：

```go
func lookup() (val string, ok bool) {
	name := "hello"
	found := true

	return name, found
}
```

只有使用**裸 `return`** 时，返回的才一定是具名返回变量当前保存的值：

```go
func lookup() (val string, ok bool) {
	val = "hello"
	ok = true
	return // 等于 return val, ok
}
```

要是什么都没赋值：

```go
func lookup() (val string, ok bool) {
	return
}
```

返回的就是它们的零值：

```go
"", false
```

所以具名返回值更像是：**函数提前帮你声明了两个可选用的返回变量，但不强迫你使用。**

### Result 适合用这个？

**可以用具名返回值来表达类似 Result 的语义，但具名返回本身并不是 Result。**

Go 最常见的“Result”写法是：

```go
func parseInt(s string) (value int, err error) {
	value, err = strconv.Atoi(s)
	return
}
```

这里：

- `value` 类似 Rust 的 `Ok(value)`
- `err` 非 `nil` 时类似 `Err(error)`

不过实际 Go 代码里，更常见的是不具名：

```go
func parseInt(s string) (int, error) {
	return strconv.Atoi(s)
}
```

调用时：

```go
value, err := parseInt("123")
if err != nil {
	fmt.Println("解析失败：", err)
	return
}

fmt.Println(value)
```

和 Rust 的逻辑可以粗略对照：

```rust
let value = parse_int("123")?;
```

Go 则是：

```go
value, err := parseInt("123")
if err != nil {
	return err
}
```

### `bool` 和 `error` 的区别

只需要表示“有没有”时，用：

```go
func findUser(id int) (user User, found bool)
```

比如 map 查询：

```go
value, ok := m[key]
```

但如果失败需要说明原因，应当用：

```go
func findUser(id int) (User, error)
```

因为 `bool` 只能表达成功或失败，不能告诉你为什么失败。

### 是否适合使用具名返回

这种签名语义确实很清楚：

```go
func findUser(id int) (user User, err error)
```

但具名返回主要改善的是**函数声明的可读性**。函数内部不一定要使用裸返回：

```go
func findUser(id int) (user User, err error) {
	user, err = repository.Find(id)
	return user, err
}
```

甚至可以直接：

```go
func findUser(id int) (user User, err error) {
	return repository.Find(id)
}
```

一般建议是：

```go
func findUser(id int) (User, error)
```

简单、直白；当返回值较多，或者返回值含义看类型不容易理解时，再具名：

```go
func pageInfo() (items []User, total int, nextCursor string, err error)
```

所以，Go 里最接近 Rust `Result<T, E>` 的不是“具名返回值”，而是：

```go
(T, error)
```

具名只是可以把它写成：

```go
(value T, err error)
```

增加语义而已。

### 它会给具名一个默认值

对。**具名返回值在函数开始时就会被创建，并自动获得对应类型的零值。**

```go
func result() (value string, ok bool, count int) {
	return
}
```

这里相当于一开始就是：

```go
value = ""
ok = false
count = 0
```

所以调用：

```go
value, ok, count := result()

fmt.Println(value) // 空字符串
fmt.Println(ok)    // false
fmt.Println(count) // 0
```

常见零值：

| 类型      | 零值                   |
| --------- | ---------------------- |
| `int`     | `0`                    |
| `float64` | `0`                    |
| `bool`    | `false`                |
| `string`  | `""`                   |
| 指针      | `nil`                  |
| 切片      | `nil`                  |
| map       | `nil`                  |
| error     | `nil`                  |
| 结构体    | 每个字段都是各自的零值 |

例如：

```go
func findUser() (user *User, err error) {
	return
}
```

默认返回：

```go
nil, nil
```

这也说明裸 `return` 要小心：如果忘记赋值，它仍然能编译，但可能默默返回零值。

```go
func divide(a, b int) (result int, err error) {
	if b == 0 {
		return // 返回 0, nil，这通常是错误的逻辑
	}

	result = a / b
	return
}
```

这里除数为零时，更合理的是显式设置错误：

```go
func divide(a, b int) (result int, err error) {
	if b == 0 {
		err = errors.New("除数不能为零")
		return
	}

	result = a / b
	return
}
```

所以，**具名返回值确实自带默认值，但不代表这些默认值在业务上一定正确。**

### 可以设不同的默认值吗

不能在函数签名里直接给具名返回值指定默认值。

这种写法不合法：

```go
func result() (value string = "hello", ok bool = true) {
	return
}
```

Go 只会自动给它们类型的零值：

```go
func result() (value string, ok bool) {
	// value 默认是 ""
	// ok 默认是 false
	return
}
```

要使用其他“默认值”，就在函数体开始时赋值：

```go
func result() (value string, ok bool) {
	value = "hello"
	ok = true

	return
}
```

也可以写成：

```go
func result() (value string, ok bool) {
	value, ok = "hello", true
	return
}
```

之后某些分支只修改其中一部分：

```go
func findName(found bool) (value string, ok bool) {
	value = "默认名字"
	ok = false

	if found {
		value = "张三"
		ok = true
	}

	return
}
```

不过很多时候，显式返回更清楚：

```go
func findName(found bool) (string, bool) {
	if found {
		return "张三", true
	}

	return "默认名字", false
}
```

所以可以自己设置初始值，但必须在函数体里赋值，不能写在返回值声明中。

## `fmt.Println(func() string { return "Hello World" }())`这种写法常用吗？return 不能省略是吧

这是一个**立即调用的匿名函数**：

```go
fmt.Println(
	func() string {
		return "Hello World"
	}(),
)
```

可以拆成两步理解：

```go
f := func() string {
	return "Hello World"
}

fmt.Println(f())
```

其中最后的 `()` 表示立刻调用这个匿名函数。

### 常用吗

Go 里会用，但不像 JavaScript 那么常见。你这个例子通常不会这样写：

```go
fmt.Println(func() string {
	return "Hello World"
}())
```

直接写更自然：

```go
fmt.Println("Hello World")
```

匿名函数立即调用更常见于需要一小段初始化逻辑时：

```go
config := func() string {
	if debug {
		return "debug"
	}
	return "production"
}()
```

或者用来限制变量作用域：

```go
func main() {
	func() {
		temp := 10
		fmt.Println(temp)
	}()

	// 这里不能访问 temp
}
```

也常和 `defer`、`go` 搭配，不过那时未必立即同步执行：

```go
defer func() {
	fmt.Println("函数结束")
}()

go func() {
	fmt.Println("并发执行")
}()
```

### `return` 能不能省略

不能。Go 不会像 Rust 那样自动把最后一个表达式作为返回值。

错误：

```go
func() string {
	"Hello World"
}()
```

必须显式写：

```go
func() string {
	return "Hello World"
}()
```

除非使用具名返回值，然后写裸 `return`：

```go
func() (result string) {
	result = "Hello World"
	return
}()
```

但这里反而更啰嗦。Go 的规则就是：**函数要返回值，就必须通过 `return` 返回，没有隐式返回最后一个表达式。**

## map 类型怎么标注

Go 里 `map` 类型写成：

```go
map[键类型]值类型
```

例如：

```go
map[string]int
```

表示：

> 键是 `string`，值是 `int` 的 map。

创建并初始化：

```go
ages := map[string]int{
	"张三": 20,
	"李四": 25,
}
```

也可以先用 `make` 创建：

```go
ages := make(map[string]int)

ages["张三"] = 20
ages["李四"] = 25
```

常见类型：

```go
map[string]string  // 字符串到字符串
map[string]int     // 字符串到整数
map[int]string     // 整数到字符串
map[string]bool    // 字符串到布尔值
map[string][]int   // 字符串到 int 切片
map[string]User    // 字符串到 User 结构体
```

作为函数参数：

```go
func printAges(ages map[string]int) {
	fmt.Println(ages)
}
```

作为返回值：

```go
func getAges() map[string]int {
	return map[string]int{
		"张三": 20,
		"李四": 25,
	}
}
```

读取值：

```go
age := ages["张三"]
```

判断键是否存在：

```go
age, ok := ages["张三"]

if ok {
	fmt.Println(age)
}
```

需要注意，下面只是声明了一个 `nil map`：

```go
var ages map[string]int
```

这时可以读取，但不能直接写入：

```go
ages["张三"] = 20 // panic
```

必须先创建：

```go
ages = make(map[string]int)
ages["张三"] = 20
```

可以简单记成：

```go
[]int            // int 切片
map[string]int   // string 到 int 的映射
```

## go 一般用什么命名法？

Go 一般使用 **驼峰命名法**，官方称为 `MixedCaps` 或 `mixedCaps`，通常不使用下划线。

```go
userName
totalCount
calculateArea
```

### 大小写还决定可见性

首字母小写：只在当前包内可见。

```go
func calculateArea() {}
var userName string
```

首字母大写：可以被其他包访问，相当于“公开”。

```go
func CalculateArea() {}
type User struct {}
var DefaultTimeout int
```

所以 Go 没有 `public`、`private` 关键字，而是靠首字母大小写控制导出。

### 常见命名

变量和函数：

```go
userName := "Tom"

func findUser() {}
func FindUser() {}
```

类型、结构体、接口：

```go
type User struct {}
type UserService struct {}
type Reader interface {}
```

常量也使用驼峰，而不是必须全大写：

```go
const MaxRetries = 3
const defaultTimeout = 10
```

Go 通常不推荐这种 Java/C 风格：

```go
const MAX_RETRIES = 3
```

### 缩写一般保持完整大写

```go
userID
httpURL
apiClient

type HTTPServer struct {}
func ServeHTTP() {}
func ParseURL() {}
```

而不是：

```go
userId
HttpServer
ParseUrl
```

特别常见的缩写包括：

```text
ID
URL
HTTP
API
JSON
SQL
TCP
IP
```

### 局部变量通常很短

作用域很小时，Go 喜欢简洁命名：

```go
for i, user := range users {
	fmt.Println(i, user)
}
```

错误值几乎固定叫：

```go
result, err := doSomething()
if err != nil {
	return err
}
```

布尔值常用：

```go
ok
found
exists
enabled
```

### 包名

包名通常全小写、简短，不用下划线：

```go
package user
package http
package config
```

一般不写：

```go
package user_service
package UserService
```

### Getter 不加 `Get`

Go 通常写：

```go
func (u User) Name() string {
	return u.name
}
```

而不是：

```go
func (u User) GetName() string {
	return u.name
}
```

Setter 仍然可以叫：

```go
func (u *User) SetName(name string) {
	u.name = name
}
```

可以先记住一句：

> Go 使用驼峰命名；首字母大写表示导出，首字母小写表示包内私有；缩写如 `ID`、`URL` 通常保持全大写。

## 你只有取一层指针的时候，才会创建上一层的指针

对，准确说是：

> 只有对一个指针变量再次取地址时，才会得到更高一层的指针值。

```go
x := 10
p := &x   // p 的类型是 *int
pp := &p  // pp 的类型是 **int
```

关系是：

```text
pp → p → x
         10
```

不过要区分两件事：

```go
&p
```

只是取得 `p` 的地址，结果类型为 `**int`。

```go
pp := &p
```

才是创建变量 `pp`，让它保存这个地址。

所以并不是“给 `p` 取地址后，系统自动生成无限多层”，而是每写一次 `&`，明确增加一层：

```go
x    // int
&x   // *int
&(&x) // 不能直接这样写，因为 &x 的结果不是可寻址变量
```

通常要通过变量逐层保存：

```go
p := &x
pp := &p
ppp := &pp
```

对应类型：

```text
int → *int → **int → ***int
```
