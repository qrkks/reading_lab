# lua 是脚本语言它的 local 变量的范围是哪里？

Lua 的 `local` 是**词法作用域变量**：从声明位置开始，一直到当前最内层代码块结束。

```lua
do
    local x = 10
    print(x) -- 10
end

print(x) -- nil，这里的 x 已经不是上面的局部变量
```

这里的“代码块”包括：

```lua
do ... end
if ... then ... end
while ... do ... end
for ... do ... end
function ... end
```

## 在函数里

```lua
local function test()
    local x = 10

    if x > 0 then
        local y = 20
        print(x, y) -- x 和 y 都可见
    end

    print(x) -- 可以
    print(y) -- nil，y 已离开作用域
end
```

需要注意：Lua 的局部变量是**从声明之后才生效**，不是整个代码块都生效。

```lua
local x = 1

do
    print(x) -- 1，使用外层 x

    local x = 2 -- 从这里开始，创建新的局部 x
    print(x) -- 2
end

print(x) -- 1
```

## 函数参数也是局部变量

```lua
local function add(a, b)
    local result = a + b
    return result
end
```

这里的 `a`、`b`、`result` 都只在函数内部可见。

## 闭包可以保存外层局部变量

```lua
local function counter()
    local n = 0

    return function()
        n = n + 1
        return n
    end
end

local count = counter()

print(count()) -- 1
print(count()) -- 2
```

虽然 `counter()` 已经执行结束，但内部函数仍然引用着 `n`，因此 `n` 不会立即消失。这种被闭包引用的局部变量在 Lua 中叫 **upvalue**。

最简单地记：

> `local` 属于它所在的最内层 `... end` 块，并且只从声明那一行之后开始有效。

不写 `local` 时，赋值通常会操作全局变量：

```lua
x = 10 -- 全局变量
local y = 20 -- 局部变量
```

所以 Lua 代码里一般应当**默认使用 `local`**，只有确实需要全局变量时才不写。

# nil 是什么的缩写？

`nil` 不是缩写，它本身就是英语单词，意思是“无、没有、零”。

在 Lua 里，`nil` 表示：

- 变量没有值
- 表中的键不存在
- 删除表中的某个键

```lua
local x = nil
print(x) -- nil

local user = {
    name = "Tom",
}

print(user.age) -- nil
```

给表字段赋 `nil`，相当于删除它：

```lua
user.name = nil
```

它和 Python 的 `None`、JavaScript 的 `null` 比较接近，但 Lua 只使用 `nil`。

# 为什么 b 打印出来是 nil?

这是 **xeus-lua 内核的执行方式造成的**，不是你的 Lua 代码写错了。

正常把这段代码保存成 `.lua` 文件运行：

```lua
a = 1
local b = 2
x, y = 1, 2

print(a, b, x, y)
```

应该输出：

```text
1    2    1    2
```

但 xeus-lua 为了实现“自动显示单元格最后一个表达式”，会把单元格拆成两部分执行：

```lua
-- 第一部分，单独执行
a = 1
local b = 2
x, y = 1, 2
```

然后再单独执行最后一行：

```lua
print(a, b, x, y)
```

它的源码确实会把代码拆成 `head` 和最后一行 `tail`，先执行 `head`，再把 `tail` 包进另一个函数执行。

而 Lua 的 `local` 变量只属于声明它的那个 **chunk（代码块/编译单元）**。第一部分执行结束后，局部变量 `b` 就不可见了；第二部分查找的是全局变量 `b`，全局环境里没有它，所以得到 `nil`。

`a`、`x`、`y` 没写 `local`，它们进入了全局环境，所以还能找到。

## 在 xeus-lua 中可以这样写

用 `do ... end` 明确包成一个完整代码块：

```lua
do
    local a = 1
    local b = 2
    local x, y = 1, 2

    print(a, b, x, y)
end
```

输出：

```text
1    2    1    2
```

或者把局部变量声明和使用写在同一行：

```lua
local b = 2
print(b)
```

所以你刚才对 `local` 的理解没有错。这里实际上是 **xeus-lua 为自动输出最后一行而拆分代码，意外破坏了顶层局部变量的作用域**。在这个 Lua Notebook 里，写多行局部代码时，最好用：

```lua
do
    -- 局部代码
end
```
