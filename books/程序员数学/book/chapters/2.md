# 二维向量绘图

## 二维向量绘图

### 三种方案：从零手写画向量（不用 vector_drawing 封装库，纯 matplotlib）

#### 一、原理

向量 = 起点 + 终点箭头；二维向量 $(x,y)$ 默认从原点 $(0,0)$ 指向 $(x,y)$。
底层只用 `matplotlib.pyplot` + `matplotlib.patches.Arrow`。

#### 二、最简完整代码（原生 matplotlib 画单个向量）

```python
import matplotlib.pyplot as plt
from matplotlib.patches import Arrow

# 1. 向量终点 (3, 2)，起点默认原点 (0,0)
vec_x, vec_y = 3, 2

# 创建画布
fig, ax = plt.subplots(figsize=(6, 6))
# 画坐标轴居中
ax.spines["left"].set_position("zero")
ax.spines["bottom"].set_position("zero")
ax.spines["right"].set_color("none")
ax.spines["top"].set_color("none")

# 绘制向量箭头：起点 (0,0)，偏移 (vec_x, vec_y)，箭头宽度 0.1
arrow = Arrow(0, 0, vec_x, vec_y, width=0.1, color="red")
ax.add_patch(arrow)

# 坐标范围，防止向量超出画布
ax.set_xlim(-5, 5)
ax.set_ylim(-5, 5)
ax.grid(True, alpha=0.3)
plt.show()
```

#### 三、封装一个自己的 draw_vector 函数（批量画多个向量）

```python
import matplotlib.pyplot as plt
from matplotlib.patches import Arrow


def draw_vector(start, end, color="blue", ax=None):
    """
    start: 起点元组 (sx, sy)
    end: 终点元组 (ex, ey)
    """
    if ax is None:
        fig, ax = plt.subplots(figsize=(6, 6))
        # 居中坐标轴
        ax.spines["left"].set_position("zero")
        ax.spines["bottom"].set_position("zero")
        ax.spines["right"].set_color("none")
        ax.spines["top"].set_color("none")
        ax.grid(True, alpha=0.3)

    sx, sy = start
    ex, ey = end
    dx = ex - sx
    dy = ey - sy
    # 画箭头
    vec = Arrow(sx, sy, dx, dy, width=0.08, color=color)
    ax.add_patch(vec)
    return ax


# 使用示例
ax = draw_vector((0, 0), (4, 2), "red")
draw_vector((0, 0), (-2, 3), "green", ax=ax)
draw_vector((1, 1), (3, 4), "orange", ax=ax)

ax.set_xlim(-5, 5)
ax.set_ylim(-5, 5)
plt.show()
```

### 加法相当于平移？

#### 结论：完全正确，二维向量加法 = 图形平移

##### 1. 几何意义

设向量 $\boldsymbol a$、$\boldsymbol b$
$\boldsymbol a + \boldsymbol b$ 两种等价理解：

1. **三角形法则**：把 $\boldsymbol b$ 的起点挪到 $\boldsymbol a$ 的终点，从 $\boldsymbol a$ 起点连到 $\boldsymbol b$ 终点，就是和向量；
2. **平移视角（你说的这个）**：
   所有点加上同一个固定向量 $\boldsymbol t$，等价于**整个图形整体平移**。

##### 2. 用恐龙例子直观理解

dino_vectors 是一堆坐标点构成恐龙轮廓：

```python
# 原始恐龙点
dino = [(6, 4), (3, 1), (1, 2), ...]
# 平移向量 t=(2,-3)：整体向右 2，向下 3
t = (2, -3)
dino_moved = [(x + t[0], y + t[1]) for x, y in dino]
```

每个点 $(x,y)\to(x+2,y-3)$，整个恐龙图案平行移动，**形状、大小、朝向完全不变**，这就是向量加法的平移效果。

##### 3. 区分关键

- **向量 + 向量**：得到新向量（三角形/平行四边形法则）
- **点坐标 + 固定向量**：图形平移（你关心的绘图场景）

##### 4. 反过来说减法

$\boldsymbol p - \boldsymbol t$ 就是**反向平移**，往相反方向挪。

##### 一句话总结

给图形里每一个坐标点加上同一个向量，本质就是整张图平移；向量加法的几何直观就是平移。

### u 点平移了 v?

分两种场景，别搞混：

#### 1. 向量相加：$u + v$

几何上有两种理解：

1. 平行四边形法则：$u$、$v$ 都从原点出发，对角线是和向量；
2. 平移视角（最贴合绘图）：
   把向量 $v$ **平移**，让 $v$ 的起点对齐 $u$ 的终点，从原点到 $v$ 新终点就是 $u+v$。

#### 2. 如果你把 $u$ 当成平面上一个点坐标

点 $u$ 加上向量 $v$：$u+v$
= 将这个点**沿着向量 $v$ 的方向平移一段距离**。
例：
点 $u=(4,3)$，平移向量 $v=(-1,1)$
$u+v=(3,4)$，点往左移 1、上移 1。

##### 一句话区分

- 点 + 向量 = 点发生平移；
- 向量 + 向量 = 把其中一个向量平移后首尾相接，合成新向量。

所以不能说“u 点平移了 v”，标准说法：
**点 $u$ 沿着向量 $v$ 平移，得到新点 $u+v$。**

### 两点之间的距离怎么计算？

#### 一、二维两点距离公式（勾股定理）

设两点：$P_1=(x_1,y_1),\ P_2=(x_2,y_2)$
两点向量差：$\vec{v}=(x_2-x_1,\ y_2-y_1)$
距离 = 这个向量的模长：

$$
d=\sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}
$$

##### 举例

$A=(4,3),\ B=(-1,1)$

$$
d=\sqrt{(-1-4)^2 + (1-3)^2}
=\sqrt{(-5)^2 + (-2)^2}
=\sqrt{25+4}
=\sqrt{29}
$$

#### 二、纯 Python 标准库代码（无第三方）

```python
import math


def distance(p1, p2):
    x1, y1 = p1
    x2, y2 = p2
    dx = x2 - x1
    dy = y2 - y1
    return math.sqrt(dx**2 + dy**2)


# 测试
print(distance((4, 3), (-1, 1)))  # sqrt(29)
```

#### 三、和点积的关系（拓展）

两点之差向量 $\boldsymbol w$，距离就是：

$$
d=\sqrt{\boldsymbol w \cdot \boldsymbol w}
$$

代码用你之前学的点积写法：

```python
import math

u = (4, 3)
v = (-1, 1)
w = [a - b for a, b in zip(u, v)]
dot = sum(i * j for i, j in zip(w, w))
d = math.sqrt(dot)
print(d)
```

#### 四、三维拓展（通用）

$P_1(x_1,y_1,z_1),\ P_2(x_2,y_2,z_2)$

$$
d=\sqrt{(x_2-x_1)^2+(y_2-y_1)^2+(z_2-z_1)^2}
$$

#### 一句话记住

两点坐标先相减，各自平方相加，最后开根号。

## 两点间距离

```python
import math


# 两点距离函数（前文定义的基础函数）
def distance(p, q):
    diff = [qi - pi for pi, qi in zip(p, q)]
    dot_self = sum(x * x for x in diff)
    return math.sqrt(dot_self)


# 多边形周长计算函数
def perimeter(vectors):
    distances = [
        distance(vectors[i], vectors[(i + 1) % len(vectors)])
        for i in range(0, len(vectors))
    ]
    return sum(distances)


# 边长为 1 的正方形测试
print(perimeter([(1, 0), (1, 1), (0, 1), (0, 0)]))
# 输出：4.0
```

### 代码逻辑拆解

1. `(i+1) % len(vectors)` 取模运算：实现**闭合多边形**，最后一个点自动连接回第一个点；
2. 列表推导遍历所有顶点，算出每一条相邻边的长度；
3. `sum(distances)` 累加所有边长得到周长；
4. 正方形 4 个顶点依次相连，4 条边长均为 1，总和 4.0。

## 平面上的角度和三角学

### 核心问题：你混淆了「角度」和「弧度」

#### 1. 为什么 `tan(45)` 不等于 1？

Python `math.tan()` 的入参单位是**弧度**，不是角度：

- `tan(45)` = 45 弧度的正切，不是 45°，结果是无理小数，无法精确表示为分数；
- 数学里 tan45°（45 度）= $\tan(\frac{\pi}{4}) = 1$，1 是精确整数（分数 $\frac{1}{1}$）。

#### 2. 正确写法（计算 45 度正切，精确等于 1）

```python
import math

# 角度转弧度：math.radians(度数)
print(math.tan(math.radians(45)))  # 输出 1.0，精确分数 1/1
```

#### 3. 能不能表示成精确分数分两种情况

##### 情况 A：三角函数自变量是**整数角度**且结果有理数（如 45°、0°、30°、60°）

可以用符号库算出精确分数：

```python
import sympy as sp

theta = sp.rad(45)  # 45 度转符号弧度 π/4
sp.pprint(theta)
print(sp.tan(theta))  # 输出 1，精确分数 1/1
```

##### 情况 B：直接 `math.tan(45)`（45 弧度，不是角度）

45 弧度不是特殊角，$\tan(45)$ 是无限不循环小数（无理数），**不能写成精确分数**，只能近似小数。

#### 4. 补充区分

- $\tan(\frac{\pi}{4}) = 1$（可精确分数 $\frac{1}{1}$）
- $\tan(45) \approx 1.619775$（无理数，无精确分数形式）

#### 一句话总结

你代码写的 `tan(45)` 是 45 弧度，没法转精确分数；
如果是 45°角的正切，结果是 1，精确分数 $\frac{1}{1}$。
分两种场景：普通浮点计算（math）、符号精确计算（sympy），两种都能写 $\pi/2$。

### 1. 标准库 math（浮点数，日常绘图用）

`math.pi` 是π的浮点近似值，直接做除法 `math.pi / 2`：

```python
import math
```

## 向量集合的变换

### 二维向量旋转完整计算（线性代数旋转矩阵）

#### 一、核心公式：旋转矩阵

设原向量点 $(x,y)$，绕原点逆时针旋转 $\theta$ 弧度，新坐标 $(x',y')$：

$$
\begin{cases}
x' = x\cos\theta - y\sin\theta \\
y' = x\sin\theta + y\cos\theta
\end{cases}
$$

矩阵形式：

$$
\begin{bmatrix}x'\\y'\end{bmatrix}
=
\begin{bmatrix}
\cos\theta & -\sin\theta \\
\sin\theta & \cos\theta
\end{bmatrix}
\begin{bmatrix}x\\y\end{bmatrix}
$$

##### 补充规则

1. $\theta>0$：**逆时针**旋转
2. $\theta<0$：**顺时针**旋转（等价 $-\theta$）
3. 所有角度必须用**弧度**计算（Python math 库要求）

#### 二、Python 基础实现（标准库，无第三方）

```python
import math


def rotate(point, theta):
    x, y = point
    c = math.cos(theta)
    s = math.sin(theta)
    x_new = x * c - y * s
    y_new = x * s + y * c
    return (x_new, y_new)


# 测试：点 (1,0) 逆时针转 90 度（π/2 弧度）
p = (1, 0)
theta = math.pi / 2
print(rotate(p, theta))  # 输出 (0, 1)，符合预期
```

#### 三、举个实例推导

点 $(1,0)$，旋转 $\theta=\frac{\pi}{2}$（90°）
$\cos(\pi/2)=0,\ \sin(\pi/2)=1$

$$
\begin{align*}
x' &= 1\cdot 0 - 0\cdot 1 = 0 \\
y' &= 1\cdot 1 + 0\cdot 0 = 1
\end{align*}
$$

结果 $(0,1)$，正确。

#### 四、绕任意点旋转（不是原点）

图形很少只绕原点转，通用三步法：

1. **平移**：把旋转中心移到原点
   $x_1 = x - cx,\quad y_1 = y - cy$
2. **旋转**：用上面旋转矩阵算 $(x_1',y_1')$
3. **平移还原**：移回原中心
   $x' = x_1' + cx,\quad y' = y_1' + cy$

封装函数：

```python
import math


def rotate_around(point, center, theta):
    x, y = point
    cx, cy = center
    # 1. 平移至原点
    x_t = x - cx
    y_t = y - cy
    # 2. 旋转
    c = math.cos(theta)
    s = math.sin(theta)
    x_r = x_t * c - y_t * s
    y_r = x_t * s + y_t * c
    # 3. 平移回去
    x_new = x_r + cx
    y_new = y_r + cy
    return (x_new, y_new)


# 测试：点 (2,0) 绕 (1,0) 转 90°
print(rotate_around((2, 0), (1, 0), math.pi / 2))
# 结果 (1, 1)
```

#### 五、和你之前知识串联

1. 平移 = 向量加法
2. 缩放 = 标量数乘
3. 旋转 = 矩阵线性变换（点积运算本质）
   旋转矩阵是正交矩阵，变换后向量长度（距离/范数）不变，只会改变方向。

#### 六、Sympy 符号精确计算（无浮点误差）

```python
import sympy as sp

x, y, θ = sp.symbols("x y θ")
c = sp.cos(θ)
s = sp.sin(θ)
x_new = x * c - y * s
y_new = x * s + y * c
print(x_new, y_new)
# 代入 θ=π/2
print(x_new.subs(θ, sp.pi / 2), y_new.subs(θ, sp.pi / 2))
```

#### 一句话记忆

逆时针旋转：x 乘 cos 减 y 乘 sin 得新 x；x 乘 sin 加 y 乘 cos 得新 y；绕非原点要先平移再旋转再还原。

### SymPy 一键旋转方案，分两种用法

#### 1. 矩阵内置旋转算子（最标准、一键）

SymPy 的 `Matrix` 自带**旋转矩阵生成函数 `rot2d(theta)`**，直接生成 2 维旋转矩阵，乘向量即完成旋转。

##### 代码示例

```python
import sympy as sp

# 定义符号
x, y = sp.symbols("x y")
vec = sp.Matrix([x, y])  # 列向量
theta = sp.pi / 2  # 旋转 90 度（π/2 弧度）

# 一键生成二维旋转矩阵（逆时针）
R = sp.rot2d(theta)
print("旋转矩阵：")
sp.pprint(R)

# 矩阵乘法 = 一键旋转
vec_rot = R * vec
print("旋转后向量：")
sp.pprint(vec_rot)

res = R * test_vec
```

##### 关键参数

`sp.rot2d(θ)`

- θ：弧度（符号/浮点数均可）
- 正θ = 逆时针旋转；负θ = 顺时针

#### 2. 绕**任意中心点**一键封装（原生无内置，自己封装一键函数）

SymPy 没有自带绕定点旋转，基于平移+rot2d 封装一键函数：

```python
import sympy as sp


def rotate_around_center(vec, center, theta):
    """
    vec: 待旋转列向量 Matrix([x,y])
    center: 旋转中心 Matrix([cx, cy])
    theta: 旋转弧度
    """
    # 平移：点 - 中心
    translated = vec - center
    # 旋转
    R = sp.rot2d(theta)
    rotated = R * translated
    # 平移复原
    return rotated + center


# 测试：点 (2,0) 绕 (1,0) 转 90°
p = sp.Matrix([2, 0])
c = sp.Matrix([1, 0])
res = rotate_around_center(p, c, sp.pi / 2)
sp.pprint(res)  # 输出 [1, 1]
```

#### 3. 纯符号推导（不用数值）

```python
import sympy as sp

x, y, θ, cx, cy = sp.symbols("x y θ cx cy")
vec = sp.Matrix([x, y])
center = sp.Matrix([cx, cy])
rot_all = rotate_around_center(vec, center, θ)
sp.pprint(rot_all)
```

会输出带 x,y,θ 的完整代数旋转公式，无浮点误差。

#### 4. 补充三维旋转

如果后续用到 3D，SymPy 还有 `rot_axis1`/`rot_axis2`/`rot_axis3` 分别绕 x/y/z 轴一键生成旋转矩阵。

#### 总结

1. **绕原点旋转：有原生一键工具 `sp.rot2d(θ)`**，乘向量直接完成旋转；
2. **绕任意点旋转**：无内置一键函数，用「平移+rot2d」封装即可实现一键调用；
3. 全程符号运算，结果精确无浮点近似，和 math 库浮点旋转区分开。

### 正交、斜交 和 旋转深度关联，分两层讲清楚

#### 一、先定义核心概念

##### 1. 正交变换（旋转、反射都属于正交变换）

满足两个硬性条件：

1. 变换矩阵 $\boldsymbol R$ 是**正交矩阵**：$\boldsymbol R^\top \boldsymbol R = \boldsymbol I$（转置等于逆矩阵）
2. 变换前后：**向量长度、向量夹角、两点距离完全不变**

二维旋转矩阵 $\boldsymbol R(\theta)=\begin{bmatrix}\cos\theta & -\sin\theta\\\sin\theta & \cos\theta\end{bmatrix}$ 天生正交：

$$
\boldsymbol R^\top \boldsymbol R
=
\begin{bmatrix}\cos\theta & \sin\theta\\-\sin\theta & \cos\theta\end{bmatrix}
\begin{bmatrix}\cos\theta & -\sin\theta\\\sin\theta & \cos\theta\end{bmatrix}
=
\begin{bmatrix}1&0\\0&1\end{bmatrix}=\boldsymbol I
$$

✅ **旋转 = 典型正交变换**

##### 2. 斜交变换（剪切变换 shear）

矩阵形式示例：

$$
\boldsymbol S=
\begin{bmatrix}1 & k\\0 & 1\end{bmatrix}
$$

它**不是正交矩阵**，不满足 $\boldsymbol S^\top \boldsymbol S=\boldsymbol I$。
斜交会**改变向量夹角**（直角不再保持直角），但**平行关系不变**。

#### 二、旋转与正交的强绑定关系

1. **所有旋转都是正交变换**
   旋转只会调转向量方向，不会拉长/缩短向量，任意两个向量的夹角、两点距离完全不变，完美符合正交定义。
2. **正交变换不只有旋转**
   正交矩阵分两类：

- $\det(\boldsymbol R)=1$：纯旋转（无镜像）
- $\det(\boldsymbol R)=-1$：旋转 + 镜面反射（翻转图形）

3. 正交变换的几何本质：保长度、保角度

#### 三、斜交和旋转完全无关，核心区别

| 特性     | 旋转（正交变换） | 斜交（剪切变换）    |
| -------- | ---------------- | ------------------- |
| 矩阵类型 | 正交矩阵         | 非正交矩阵          |
| 长度     | 向量长度不变     | 向量长度改变        |
| 直角     | 直角永远保持直角 | 直角会变成锐角/钝角 |
| 面积     | 图形面积不变     | 图形面积不变        |
| 几何效果 | 图形原地转圈     | 图形平行拉扯、变形  |

举直观例子：
正方形旋转任意角度，四个角依旧 90°；
正方形做斜交剪切，直接变成平行四边形，直角消失。

#### 四、坐标系层面的正交/斜交（拓展，对应你笛卡尔坐标）

1. **正交坐标系（直角坐标系）**
   基向量 $\boldsymbol e_1,\boldsymbol e_2$ 互相垂直，点积 $\boldsymbol e_1\cdot\boldsymbol e_2=0$，我们平时用的 xy 笛卡尔坐标系就是正交系；
   把正交坐标系整体旋转任意角度，得到的新坐标系**依然是正交坐标系**（旋转保垂直）。

2. **斜交坐标系**
   基向量不垂直，点积不为 0；
   **任何旋转都不可能把正交坐标系变成斜交坐标系**，因为旋转永远保留垂直关系。
   想要得到斜交坐标系，必须用剪切（斜交变换），不能靠旋转。

#### 五、一句话总结

1. 旋转属于**正交变换**，二者强绑定，旋转永远保长度、保垂直；
2. 斜交是完全独立的变形，不属于正交变换，和旋转没有任何包含关系；
3. 旋转无法产生斜交效果，斜交也不能通过旋转消除。

## 用 Matplotlib 绘图

```python
import matplotlib.pyplot as plt

# 示例配套类（简化版，匹配代码逻辑）
class Points:
    def __init__(self, vectors, color="black"):
        self.vectors = vectors  # 存储一组坐标向量 [(x1,y1), (x2,y2)...]
        self.color = color      # 点的颜色

def draw(*objects):
    # 省略：画布、坐标轴、样式初始化代码
    for obj in objects:
        # 判断当前遍历到的对象类型
        elif type(obj) == Points:
            # 提取所有点的 x 坐标
            xs = [v[0] for v in obj.vectors]
            # 提取所有点的 y 坐标
            ys = [v[1] for v in obj.vectors]
            # matplotlib 散点图绘制所有点，使用对象自带颜色
            plt.scatter(xs, ys, color=obj.color)
    # 省略：展示画布 plt.show()
```

#### 逐段解释

1. **`def draw(*objects):`**
   `*objects` 可变位置参数，可以一次性传入多个绘图对象，例如 `draw(points1, points2, vector1)`，函数内部会把所有传入对象打包成元组循环遍历。

2. **`for obj in objects:`**
   循环遍历你传给 `draw()` 的每一个图形对象（点集、向量、多边形等），分支判断分别处理不同图形。

3. **`elif type(obj) == Points:`**
   判断当前对象是自定义 `Points` 点集合类，专门处理批量坐标点绘制；
   - `obj.vectors`：实例属性，存放所有二维坐标向量列表 `[(x0,y0), (x1,y1)...]`
   - `obj.color`：实例属性，记录这批点统一使用的颜色

4. **列表推导提取坐标**

   ```python
   xs = [v[0] for v in obj.vectors]  # 取出每个向量第一个元素（x 轴）
   ys = [v[1] for v in obj.vectors]  # 取出每个向量第二个元素（y 轴）
   ```

   例：`vectors = [(1,0), (1,1), (0,1)]` → `xs=[1,1,0]`，`ys=[0,1,1]`

5. **`plt.scatter(xs, ys, color=obj.color)`**
   Matplotlib 散点绘图 API，一次性批量绘制全部坐标点，复用对象预设颜色。

```python
import matplotlib.pyplot as plt


class Points:
    def __init__(self, vectors, color="black"):
        self.vectors = vectors
        self.color = color


def draw(*objects):
    plt.figure(figsize=(6, 6))
    plt.axhline(0, color="gray", lw=0.8)
    plt.axvline(0, color="gray", lw=0.8)
    plt.grid(alpha=0.3)

    for obj in objects:
        if type(obj) == Points:
            xs = [v[0] for v in obj.vectors]
            ys = [v[1] for v in obj.vectors]
            plt.scatter(xs, ys, color=obj.color, s=60)

    plt.axis("equal")
    plt.show()


# 测试：正方形四个顶点红点
square_points = Points([(1, 0), (1, 1), (0, 1), (0, 0)], color="red")
draw(square_points)
```
