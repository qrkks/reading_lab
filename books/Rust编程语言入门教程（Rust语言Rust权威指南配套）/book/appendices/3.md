# 3

## loop，while 什么区别

区别很简单：

> `loop` 是“无条件一直循环”；`while` 是“条件为真才继续循环”。

### `loop`

```rust
loop {
    println!("一直执行");
}
```

它会无限循环，除非内部遇到：

```rust
break;
```

例如：

```rust
let mut x = 0;

loop {
    dbg!(x);
    x += 1;

    if x == 3 {
        break;
    }
}
```

### `while`

```rust
let mut x = 0;

while x < 3 {
    dbg!(x);
    x += 1;
}
```

每轮开始前检查条件：

```text
x < 3 吗？
├── 是 → 执行循环体
└── 否 → 结束循环
```

所以这两段逻辑大致等价：

```rust
while x < 3 {
    x += 1;
}
```

```rust
loop {
    if x >= 3 {
        break;
    }

    x += 1;
}
```

### Rust 里一个很特别的区别

`loop` 可以通过 `break` 返回值，所以它可以是一个真正有用的表达式：

```rust
let result = loop {
    break 10;
};

dbg!(result);
```

这里：

```rust
break 10;
```

不只是结束循环，还让整个 `loop` 的值变成 `10`。

因此：

```rust
let result = loop {
    // 做一些事情
    break 10;
};
```

有点像一个会反复执行、最后才算出结果的表达式。

而 `while` 通常不能这样返回一个普通值：

```rust
let result = while condition {
    // ...
};
```

`while` 的结果一般是 `()`，它主要用于控制流程。

### 怎么选

条件天然存在时，用 `while`：

```rust
while number > 0 {
    number -= 1;
}
```

不知道要循环多少次，需要在内部决定何时退出时，用 `loop`：

```rust
loop {
    let input = read_input();

    if input == "quit" {
        break;
    }
}
```

可以这样记：

```text
while = 条件控制循环
loop  = 内部用 break 控制循环
```

而 Rust 的 `loop` 还有额外能力：

```text
break value
```

可以让整个循环产生一个值。

## loop 是每轮结束配合 break 检查；while 是每轮开始前检查；loop 很像 while true 啊？

对，`loop` 很像：

```rust
while true {
    // ...
}
```

但你前一句要稍微修正：

> `loop` 不是固定在“每轮结束时检查”，而是循环体内走到 `break` 的地方才检查并退出。

例如可以在开头检查：

```rust
loop {
    if finished {
        break;
    }

    do_work();
}
```

也可以在中间或结尾检查：

```rust
loop {
    do_first();

    if finished {
        break;
    }

    do_second();
}
```

因此更准确地说：

- `while`：检查位置固定在每轮开始前
- `loop`：没有自动条件检查，退出条件由你放在循环体中的任意位置

### 是否至少执行一次

`while` 可能一次都不执行：

```rust
let x = 10;

while x < 5 {
    println!("不会执行");
}
```

`loop` 一定会先进入循环体：

```rust
loop {
    println!("至少执行一次");
    break;
}
```

所以有时 `loop` 也能模拟其他语言的 `do...while`：

```rust
loop {
    do_work();

    if !condition {
        break;
    }
}
```

这相当于：

```text
先执行一次
→ 再检查是否继续
```

### 为什么 Rust 还专门提供 `loop`

虽然控制流程上它很像 `while true`，但 `loop` 更明确地表达：

> 这是一个主动依靠 `break` 退出的循环。

而且 `loop` 可以返回值：

```rust
let result = loop {
    let value = calculate();

    if value > 10 {
        break value;
    }
};
```

这里 `result` 得到 `value`。

而 `while` 不能这样：

```rust
// 不允许通过 break 6 返回值
while condition {
    break 6;
}
```

因此可以记成：

```text
while 条件 {
    ...
}
```

是“外部条件控制”。

```text
loop {
    ...
    if 条件 {
        break;
    }
}
```

是“循环体内部控制”。

所以你的直觉基本正确：**`loop` 就是 Rust 中更正式、更强的 `while true`。**

## arr.iter() 是什么？迭代器？

对，`arr.iter()` 就是：

> **为数组 `arr` 创建一个迭代器，按顺序借用其中的元素。**

例如：

```rust
let arr = [10, 20, 30];

let iter = arr.iter();
```

这里：

```rust
arr.iter()
```

返回的不是新数组，而是一个迭代器，大致类型是：

```rust
std::slice::Iter<'_, i32>
```

它内部记录“当前走到数组的哪个位置”。官方文档将其描述为遍历切片元素的迭代器。

### 手动调用 `next()`

```rust
let arr = [10, 20, 30];
let mut iter = arr.iter();

dbg!(iter.next()); // Some(&10)
dbg!(iter.next()); // Some(&20)
dbg!(iter.next()); // Some(&30)
dbg!(iter.next()); // None
```

`next()` 每调用一次，就取出下一个元素：

```text
Some(&10)
Some(&20)
Some(&30)
None
```

这里是 `&10`，不是 `10`，因为 `.iter()` **不会拿走元素，只会借用元素**。因此迭代项的类型是：

```rust
&i32
```

而不是：

```rust
i32
```

### 在 `for` 中使用

```rust
let arr = [10, 20, 30];

for x in arr.iter() {
    dbg!(x);
}
```

此时：

```rust
x: &i32
```

`for` 会在背后不断调用迭代器的 `next()`，直到得到 `None`。`Iterator` 的核心就是反复通过 `next()` 产生 `Some(...)`，最后产生 `None`。

而且下面两种写法基本等价：

```rust
for x in arr.iter() {
    dbg!(x);
}
```

```rust
for x in &arr {
    dbg!(x);
}
```

官方文档也说明，遍历切片时可以省略显式的 `.iter()`。

### 三种常见迭代方式

```rust
arr.iter()      // 借用元素，得到 &T
arr.iter_mut()  // 可变借用元素，得到 &mut T
arr.into_iter() // 取得元素的所有权，得到 T
```

例如修改元素：

```rust
let mut arr = [10, 20, 30];

for x in arr.iter_mut() {
    *x *= 2;
}

dbg!(arr); // [20, 40, 60]
```

所以你现在先记：

> `arr.iter()` 创建一个“不拿走数组、只逐个借用元素”的迭代器。

## 为什么放到大括号里能够运行？

因为大括号 `{ ... }` 在 Rust 里会创建一个**新的局部作用域**。

```rust
{
    let arr = [10, 20, 30];
    let mut iter = arr.iter();

    dbg!(iter.next());
}
```

变量的生命周期变成：

```text
进入 {
  创建 arr
  创建 iter，iter 借用 arr
  使用 iter
  先销毁 iter
  再销毁 arr
}
离开 }
```

编译器可以明确保证：

> `iter` 一定会在 `arr` 被销毁之前结束。

所以借用关系是安全的。

而在 Evcxr 顶层写：

```rust
let arr = [10, 20, 30];
let iter = arr.iter();
```

Evcxr 会尝试把顶层变量保存到下一个代码单元中：

```text
当前代码框
arr、iter
    ↓ 持久保存
下一个代码框继续使用
```

问题是 `iter` 内部借用了 `arr`。Evcxr 不适合把这种带有非静态引用的顶层变量长期保存，所以报错。

大括号相当于告诉 Evcxr：

> 这些变量只是临时变量，执行完这个块就销毁，不要保存到后面的代码框。

也可以直观地试：

```rust
{
    let arr = [10, 20, 30];
    let iter = arr.iter();
}

dbg!(iter); // 报错：这里已经找不到 iter
```

因为 `iter` 只存在于大括号内部。

因此，不是大括号让引用“变成了静态引用”，而是它**缩短了引用的生命周期**，让 `iter` 不需要跨 Evcxr 单元持久存在。

## 除了 R 是不是基本上所有的语言 range 都是左闭右开

不算。更准确地说：

> **左闭右开是很多系统语言和标准库特别偏爱的设计，但远不是“除了 R 都这样”。**

Rust 同时提供两种：

```rust
0..5   // 左闭右开：0, 1, 2, 3, 4
0..=5  // 两端都闭：0, 1, 2, 3, 4, 5
```

常见语言大致是：

| 语言   | 写法          | 是否包含右端点 |
| ------ | ------------- | -------------- |
| Python | `range(0, 5)` | 不包含 5       |
| Rust   | `0..5`        | 不包含 5       |
| Rust   | `0..=5`       | 包含 5         |
| Ruby   | `0...5`       | 不包含 5       |
| Ruby   | `0..5`        | 包含 5         |
| Kotlin | `0 until 5`   | 不包含 5       |
| Kotlin | `0..5`        | 包含 5         |
| Swift  | `0..<5`       | 不包含 5       |
| Swift  | `0...5`       | 包含 5         |
| Julia  | `0:5`         | 包含 5         |
| R      | `0:5`         | 包含 5         |

JavaScript、Go、C 本身没有一个像 Rust/Python 那样统一的内建 `range` 写法，通常只是循环习惯上写成：

```c
for (int i = 0; i < 5; i++)
```

这也是左闭右开。

左闭右开流行，是因为它特别适合索引：

```text
长度 = 结束位置 - 开始位置
```

例如：

```rust
let arr = [10, 20, 30, 40, 50];

&arr[1..4]
```

得到索引 `1、2、3`，元素数量刚好是：

```text
4 - 1 = 3
```

而且相邻区间很好拼：

```text
0..5
5..10
```

不会重复，也不会漏掉边界。

所以你的直觉有一半对：

> **在数组索引、切片和循环中，左闭右开确实是主流约定；但不少语言同时提供闭区间，甚至默认就是闭区间。**

## 这方便啊，右闭右开可以控制

对，Rust 是**右端点开不开可以直接控制**：

```rust
1..5    // [1, 5)：1, 2, 3, 4
1..=5   // [1, 5]：1, 2, 3, 4, 5
```

还可以省略某一端：

```rust
..5     // 小于 5
..=5    // 小于等于 5
1..     // 大于等于 1
..      // 整个范围
```

所以 Rust 的符号很直观：

```text
..   右边不包含
..=  右边包含
```

不过左端点只要写出来，默认就是包含的。Rust 没有直接表示“左开右闭”的特殊语法；整数场景可以把起点加一：

```rust
2..=5 // 相当于整数上的 (1, 5]
```

你刚才说的应该是：**右开、右闭可以控制**。这确实比只提供一种 `range` 灵活。
