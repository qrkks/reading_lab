# rust 的枚举是干什么用的？

Rust 的枚举不只是“几个整数常量”，而是用来表达：

> **一个值只能处于若干种情况中的一种，而且每种情况还能携带不同的数据。**

这正好对应你刚才问 Go 的：

```text
返回值能不能是 A 或 B？
```

Rust 可以直接写：

```rust
enum Output {
    A(A),
    B(B),
}
```

意思是：

```text
Output 要么是 A
要么是 B
不可能同时是两者
也不可能是其他类型
```

## 最普通的枚举

```rust
enum Direction {
    Up,
    Down,
    Left,
    Right,
}
```

使用：

```rust
let direction = Direction::Up;
```

这部分和其他语言的枚举差不多。

但 Rust 枚举真正厉害的是，**每个分支可以带数据**：

```rust
enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(u8, u8, u8),
}
```

这四种情况的数据结构完全不同：

```rust
let a = Message::Quit;

let b = Message::Move { x: 10, y: 20 };

let c = Message::Write(String::from("hello"));

let d = Message::ChangeColor(255, 0, 0);
```

可以把它理解成：

```text
Message =
    Quit
    或 Move { x, y }
    或 Write(String)
    或 ChangeColor(r, g, b)
```

## 必须用 `match` 处理不同情况

```rust
fn handle(message: Message) {
    match message {
        Message::Quit => {
            println!("退出");
        }

        Message::Move { x, y } => {
            println!("移动到 {x}, {y}");
        }

        Message::Write(text) => {
            println!("文本：{text}");
        }

        Message::ChangeColor(r, g, b) => {
            println!("颜色：{r}, {g}, {b}");
        }
    }
}
```

Rust 编译器会检查你有没有漏掉某个分支。

比如漏掉 `Quit`：

```rust
match message {
    Message::Write(text) => println!("{text}"),
}
```

编译器会直接报错，因为 `message` 还可能是其他情况。

这就是枚举非常安全的地方：

> 所有可能状态都被明确列出，处理时编译器强迫你考虑完整。

## `Option` 就是枚举

Rust 没有普通意义上的 `null`，而是使用：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

例如查找用户：

```rust
fn find_user(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Tom"))
    } else {
        None
    }
}
```

调用：

```rust
match find_user(1) {
    Some(name) => println!("找到用户：{name}"),
    None => println!("没有找到"),
}
```

它表达的是：

```text
要么有一个 String
要么没有值
```

在 Go 里你可能写：

```go
func FindUser(id int) (*User, error)
```

然后靠 `nil` 表示没有用户。

Rust 则把“有值”和“没值”明确变成两种枚举状态。

## `Result` 也是枚举

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

例如：

```rust
fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err(String::from("不能除以零"))
    } else {
        Ok(a / b)
    }
}
```

返回结果只能是：

```text
Ok(计算结果)
或
Err(错误信息)
```

处理：

```rust
match divide(10, 2) {
    Ok(value) => println!("结果：{value}"),
    Err(error) => println!("错误：{error}"),
}
```

所以 Rust 的错误不是额外返回值，而是：

> 函数返回了一个值，这个值要么代表成功，要么代表失败。

## 枚举还能表达业务状态

例如订单：

```rust
enum OrderStatus {
    Pending,
    Paid { paid_at: String },
    Shipped { tracking_number: String },
    Cancelled { reason: String },
}
```

这样每种状态只能拥有与自己有关的数据：

```rust
let status = OrderStatus::Shipped {
    tracking_number: String::from("SF123456"),
};
```

相比用一个结构体：

```rust
struct OrderStatus {
    status: String,
    paid_at: Option<String>,
    tracking_number: Option<String>,
    cancel_reason: Option<String>,
}
```

后者可能产生很多非法组合：

```text
状态是 Pending，但 tracking_number 却有值
状态是 Cancelled，但 paid_at 和 tracking_number 都有值
```

枚举则从类型层面阻止这些非法状态。

## 和 Go 接口的区别

Go 里可能这样模拟：

```go
type Result interface {
	isResult()
}
```

然后让 `A`、`B` 实现它。

但接口表达的是：

> 任何具有这些方法的类型都可以进来。

Rust 枚举表达的是：

> 只有我明确列出的这些分支可以存在。

因此：

```rust
enum Output {
    A(A),
    B(B),
}
```

是一个**封闭集合**。以后不会突然有别的类型悄悄成为 `Output`。

可以把它概括为：

```text
struct：一个值同时包含哪些字段
enum：一个值可能是哪一种情况
trait：一个类型能够做什么
```

例如：

```rust
struct User {
    name: String,
    age: u8,
}
```

表示用户同时有名字和年龄。

```rust
enum LoginResult {
    Success(User),
    WrongPassword,
    UserNotFound,
}
```

表示登录结果只能是三种情况之一。

这就是 Rust 枚举最核心的用途：**把“可能发生的不同情况”直接建模进类型系统里。**

# option 也是枚举类型吗？

对，**`Option` 就是枚举类型**。

Rust 标准库里它本质上差不多就是这样：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

意思是：

- `Some(T)`：有一个值
- `None`：没有值

所以比如：

```rust
let a = Some(5);
let b: Option<i32> = None;
```

这里：

- `a` 的类型是 `Option<i32>`
- `b` 的类型也是 `Option<i32>`

## 为什么它很重要

它是在类型层面表达：

> 这个值“可能存在，也可能不存在”。

比如查找用户：

```rust
fn find_user(id: u32) -> Option<String> {
    if id == 1 {
        Some(String::from("Tom"))
    } else {
        None
    }
}
```

这就比很多语言里的 `null` 更清楚，也更安全。

---

## 怎么用

最标准的是 `match`：

```rust
match find_user(1) {
    Some(name) => println!("找到用户：{}", name),
    None => println!("没找到"),
}
```

也常用 `if let`：

```rust
if let Some(name) = find_user(1) {
    println!("找到用户：{}", name);
}
```

---

## 你可以把它理解成

```text
Option<T> = 有一个 T，或者没有
```

这其实正是枚举最擅长的事：

> **表达“一个值只能是几种情况之一”**

而 `Option<T>` 就是最经典的例子。

---

## 顺便一提

你前面看到的 `Result` 也是枚举：

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

所以可以顺手记成：

- `Option<T>`：有 / 没有
- `Result<T, E>`：成功 / 失败

它们都是 Rust 里最核心的枚举类型。

如果你愿意，我下一条可以继续给你讲：

> **`Option<T>` 和 Go 里的 `nil`、`(T, bool)`、`(T, error)` 分别怎么对应。**

# 这段讲的什么意思？

这段是在解释：

> **Rust 不允许普通变量在“正常值”和 `null` 之间偷偷切换，而是要求你用 `Option<T>` 明确表示“可能没有值”。**

逐句看。

## 1. 其他语言中的 `null`

比如某些语言里：

```java
String name = null;
```

变量 `name` 的类型明明是 `String`，但它实际上可能处于两种状态：

```text
一个真正的 String
或者 null
```

问题在于，代码只看到 `String` 类型时，不一定能立刻意识到它可能为空：

```java
name.length();
```

如果 `name` 是 `null`，运行时就会出错。

这类错误在不同语言里叫：

- NullPointerException
- nil pointer dereference
- undefined is not an object

也就是幻灯片说的：

> 当你像使用正常值一样使用 `null` 时，就会引发错误。

---

## 2. “Billion Dollar Mistake”是什么意思

`null` 的发明者 Tony Hoare 后来把引入空引用称为：

> “十亿美元的错误。”

不是说真的精确损失了十亿美元，而是说几十年来，空指针造成了大量：

- 程序崩溃
- 安全漏洞
- 调试成本
- 生产事故

幻灯片中的 `Billion Dollar Mistake` 就是在引用这个说法。

---

## 3. 但“没有值”这个需求确实存在

`null` 的问题不是“没有值”这个概念没用。

例如：

```text
数据库里可能查不到用户
搜索可能找不到结果
配置项可能不存在
树节点可能没有子节点
```

我们确实需要表达：

> 这里可能有一个值，也可能没有。

幻灯片最后一句“Null 的概念还是有用的”，说的就是这个。

Rust 没有删除“缺失值”这个概念，而是换了一种更明确的表达方式。

---

## 4. Rust 用 `Option<T>` 表示

```rust
enum Option<T> {
    Some(T),
    None,
}
```

例如：

```rust
let name: Option<String> = Some(String::from("Tom"));
```

或者：

```rust
let name: Option<String> = None;
```

这里类型直接告诉你：

```text
name 可能包含一个 String
也可能没有 String
```

因此，你不能直接把 `Option<String>` 当作 `String` 使用：

```rust
let length = name.len(); // 编译错误
```

必须先处理两种情况：

```rust
match name {
    Some(value) => println!("长度是 {}", value.len()),
    None => println!("没有名字"),
}
```

Rust 编译器会逼着你考虑 `None`，而不是等程序运行时突然崩溃。

---

## 5. `String` 和 `Option<String>` 是两个不同类型

这是关键区别。

在允许 `null` 的语言中，可能是：

```text
String 实际上可能是 String 或 null
```

而在 Rust 中：

```text
String         一定有 String
Option<String> 可能有，也可能没有
```

因此：

```rust
fn print_name(name: String) {
    println!("{name}");
}
```

调用这个函数时，`name` 一定存在。

而：

```rust
fn print_name(name: Option<String>) {
    match name {
        Some(value) => println!("{value}"),
        None => println!("没有名字"),
    }
}
```

类型签名明确告诉调用者和函数作者：这里必须处理缺失情况。

## 更准确地说

幻灯片标题“Rust 没有 Null”稍微简化了。

准确说是：

> **Rust 的普通安全类型和普通引用不能是 `null`。**

例如：

```rust
let x: &String;
```

一个 `&String` 必须指向有效的 `String`，不能是空引用。

不过 Rust 的底层裸指针：

```rust
*const String
*mut String
```

技术上仍然可以为空，但那主要用于底层代码、FFI 和 `unsafe`，日常安全 Rust 不靠它表达缺失值。

所以这张幻灯片的核心就是：

```text
其他语言：
String = String 或 null

Rust：
String = 一定有值
Option<String> = Some(String) 或 None
```

`Option` 不是把 `null` 换了个名字，而是把“有值或无值”正式建模成了一个枚举，并要求编译器检查你是否处理完整。

# Some 是什么？

`Some` 是 `Option<T>` 这个枚举的一个**枚举分支**，表示：

> **这里有一个值。**

标准库中的 `Option` 大致定义为：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

其中：

```rust
Some(T)
```

表示“有一个类型为 `T` 的值”；而：

```rust
None
```

表示“没有值”。

例如：

```rust
let age = Some(18);
```

这里不是说 `age` 等于整数 `18`，而是：

```text
age 是一个 Option<i32>
它当前处于 Some 分支
Some 里面装着整数 18
```

可以粗略想成一个盒子：

```text
Some(18)
┌──────────┐
│    18    │
└──────────┘
```

而：

```rust
let age: Option<i32> = None;
```

就是：

```text
这是一个可能装 i32 的 Option
但当前没有装值
```

## `Some` 也像一个构造函数

对于这个枚举：

```rust
enum Option<T> {
    Some(T),
    None,
}
```

`Some` 可以接收一个 `T`，构造出一个 `Option<T>`：

```rust
Some(18)                   // Option<i32>
Some("hello")              // Option<&str>
Some(String::from("Tom"))  // Option<String>
```

所以类型变化是：

```text
18          是 i32
Some(18)    是 Option<i32>
```

这两个不是同一种类型。

## 怎么取出 `Some` 里的值

用 `match`：

```rust
let age = Some(18);

match age {
    Some(value) => println!("年龄是 {value}"),
    None => println!("没有年龄"),
}
```

这里：

```rust
Some(value)
```

是在做模式匹配。如果 `age` 是 `Some(18)`，那么里面的 `18` 就会被取出来，绑定给 `value`。

也可以使用 `if let`：

```rust
if let Some(value) = age {
    println!("年龄是 {value}");
}
```

一句话记忆：

```text
Some(value)：有值，并把值装在里面
None：没有值
Option<T>：Some(T) 或 None
```

# 啥意思，Some 是推断传入的数据类型的？反查？

对，你可以先把 `Some` 粗略理解成一个**泛型包装器**：

```rust
Some(值)
```

编译器会根据里面的值，推断 `Option` 装的是什么类型。

例如：

```rust
let x = Some(18);
```

推断过程大致是：

```text
18 是 i32
    ↓
Some 里面装的是 i32
    ↓
x 的类型是 Option<i32>
```

所以确实有一点你说的“从传入数据反查外层类型”的感觉。

## `Some` 大致像这个函数

概念上可以把它看成：

```rust
fn Some<T>(value: T) -> Option<T>
```

当然，`Some` 实际上是枚举分支，不是普通函数；只是它使用起来很像函数。

```rust
Some(18)       // Option<i32>
Some("hello")  // Option<&str>
Some(true)     // Option<bool>
```

## 也可以反过来由外层类型决定里面的类型

```rust
let x: Option<u8> = Some(18);
```

这里编译器先看到：

```text
x 必须是 Option<u8>
```

于是知道 `Some` 里面必须是 `u8`，数字常量 `18` 就被确定成 `u8`。

所以 Rust 的类型推断不只是单向的，而是会结合两边的信息：

```text
Some(18)
   ↑ 由里面推断外面

let x: Option<u8> = Some(18);
       ↓ 由外面约束里面
```

## 为什么 `None` 常常需要标注类型

`None` 里面没有值，所以编译器没法从内部推断 `T`：

```rust
let x = None; // 不知道是 Option<i32> 还是 Option<String>
```

需要告诉它：

```rust
let x: Option<i32> = None;
```

或者：

```rust
let x = None::<i32>;
```

因此区别是：

```rust
Some(18) // 有内部数据，可以推断 T
None     // 没有内部数据，通常需要上下文确定 T
```

所以你的理解基本对：**`Some` 根据放进去的值推断 `T`，再得到整个 `Option<T>` 的类型；同时也可以根据外部要求反过来确定内部类型。**

# 噢，是不是用 option 的时候，变量的值就得用 some 包起来？

对。

当变量类型是：

```rust
Option<T>
```

它的值就只能是这个枚举的两个分支之一：

```rust
Some(value)
None
```

例如：

```rust
let age: Option<i32> = Some(18);
let unknown_age: Option<i32> = None;
```

不能直接写：

```rust
let age: Option<i32> = 18; // 错误
```

因为：

```text
18       是 i32
Some(18) 是 Option<i32>
```

你可以把 `Some` 理解成：**把普通值包装成“可能存在的值”**。

```rust
let name: String = String::from("Tom");

let optional_name: Option<String> = Some(name);
```

不过，如果某个函数本来就返回 `Option<T>`，就不用再包一次：

```rust
fn find_number() -> Option<i32> {
    Some(5)
}

let number: Option<i32> = find_number();
```

这里 `find_number()` 返回的已经是 `Option<i32>` 了。

一句话记住：

```text
Option<T> 的值只能是 Some(T) 或 None
```

# 箭头符号是干什么的？

在 Rust 里常见有两种“箭头”，作用不同。

### `->`：声明函数返回类型

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

这里：

```rust
-> i32
```

表示这个函数会返回一个 `i32`。

可以读成：

```text
接收两个 i32 → 返回一个 i32
```

如果函数不返回值，就通常不写：

```rust
fn say_hello() {
    println!("hello");
}
```

它实际上返回的是单元类型 `()`。

返回 `Option` 时：

```rust
fn find_number() -> Option<i32> {
    Some(5)
}
```

意思是函数返回：

```text
Option<i32>
```

也就是 `Some(i32)` 或 `None`。

### `=>`：`match` 分支的“对应结果”

```rust
match x {
    Some(value) => println!("{value}"),
    None => println!("没有值"),
}
```

这里可以读成：

```text
匹配到 Some(value) → 执行右边
匹配到 None        → 执行右边
```

所以记住：

```text
->  函数返回什么
=>  match 分支做什么
```

# 传入的类型如果是 Enum 那就只限定在它的分支里取一个了

对，理解得对。

函数参数如果声明成某个枚举类型：

```rust
enum Direction {
    Up,
    Down,
    Left,
    Right,
}

fn walk(direction: Direction) {
    // ...
}
```

那么传进去的值只能是 `Direction` 的某一个分支：

```rust
walk(Direction::Up);
walk(Direction::Left);
```

不能传其他类型：

```rust
walk("Up"); // 错误：&str 不是 Direction
walk(1); // 错误：整数不是 Direction
```

而且一个枚举值在某一时刻只能处于一个分支：

```text
Direction::Up
或者 Direction::Down
或者 Direction::Left
或者 Direction::Right
```

不会同时是两个分支。

分支还可以携带数据：

```rust
enum Message {
    Quit,
    Write(String),
    Move { x: i32, y: i32 },
}

fn handle(message: Message) {
    match message {
        Message::Quit => println!("退出"),
        Message::Write(text) => println!("{text}"),
        Message::Move { x, y } => println!("{x}, {y}"),
    }
}
```

传参时仍然只能从这些分支里选一个：

```rust
handle(Message::Quit);
handle(Message::Write(String::from("hello")));
handle(Message::Move { x: 10, y: 20 });
```

`Option<i32>` 也是同样的逻辑：

```rust
fn show(value: Option<i32>) {
    // ...
}

show(Some(5));
show(None);
```

只能传：

```text
Some(i32)
或
None
```

不能直接传：

```rust
show(5); // 错误：5 是 i32，不是 Option<i32>
```

所以可以这样记：

> `struct` 限定一个值必须同时拥有哪些字段；  
> `enum` 限定一个值只能从哪些情况中选择一个。

# 一个分支就是值，一个 Enum 不是值

对，基本可以这么理解：

> **`enum` 是类型；某个具体分支构造出来的东西，才是这个类型的值。**

例如：

```rust
enum Direction {
    Up,
    Down,
}
```

这里：

```rust
Direction
```

是一个**类型**，不是值。

而：

```rust
Direction::Up
Direction::Down
```

是两个 `Direction` 类型的值：

```rust
let a: Direction = Direction::Up;
let b: Direction = Direction::Down;
```

这和结构体很像：

```rust
struct User {
    name: String,
}
```

其中：

```rust
User
```

是类型，而：

```rust
User {
    name: String::from("Tom"),
}
```

才是一个值。

## 带数据的枚举分支稍微特殊

```rust
enum Message {
    Quit,
    Write(String),
}
```

这里：

```rust
Message::Quit
```

不需要额外数据，所以它本身直接是一个值。

但：

```rust
Message::Write
```

还不是完整的 `Message` 值，它更像一个构造器，需要传入 `String`：

```rust
let message = Message::Write(String::from("hello"));
```

最终这个才是值：

```rust
Message::Write(String::from("hello"))
```

可以这样归纳：

```text
Message                          枚举类型
Message::Quit                    一个 Message 值
Message::Write                   分支构造器
Message::Write("hello".into())   一个 Message 值
```

`Option` 也是一样：

```text
Option<i32>  类型
Some         分支构造器
Some(5)      Option<i32> 的值
None         Option<i32> 的值
```

所以你这句话更精确地说是：

> **枚举是类型；枚举分支用于构造这个类型的值。无数据的分支本身就可以直接作为值。**

# 我要传借用，函数的参数就得定义成借用啊？

对，**普通函数里，调用时传什么类型，参数就必须声明成对应类型。**

传值：

```rust
fn value_in_cents(coin: Coin) -> u8 {
    // ...
}

value_in_cents(Coin::Penny);
```

参数和实参都是：

```text
Coin
```

传不可变借用：

```rust
fn value_in_cents(coin: &Coin) -> u8 {
    // ...
}

let coin = Coin::Penny;
value_in_cents(&coin);
```

参数和实参都是：

```text
&Coin
```

需要修改时，传可变借用：

```rust
fn change_coin(coin: &mut Coin) {
    *coin = Coin::Dime;
}

let mut coin = Coin::Penny;
change_coin(&mut coin);
```

对应关系就是：

```text
coin       → Coin
&coin      → &Coin
&mut coin  → &mut Coin
```

函数签名决定调用者应该怎么传：

```rust
fn f(x: Coin)       // 获取所有权
fn f(x: &Coin)      // 临时读取
fn f(x: &mut Coin)  // 临时读取并修改
```

不过**方法调用**有一点特殊，Rust 经常会自动借用：

```rust
struct User {
    name: String,
}

impl User {
    fn show(&self) {
        println!("{}", self.name);
    }
}

let user = User {
    name: String::from("Tom"),
};

user.show();
```

虽然方法参数是 `&self`，调用时不用手动写：

```rust
(&user).show();
```

Rust 会自动借用。但普通函数通常不会替你把：

```rust
Coin
```

自动变成：

```rust
&Coin
```

所以你可以先记住：

> 普通函数严格按参数类型传；方法的 `self` 经常会自动借用或解引用。

# dbg! 那种既可以传借用，又可以传值的是怎么定义的？

因为 `dbg!` **不是普通函数，而是宏**。

它接收的是一个“表达式”：

```rust
dbg!(表达式)
```

所以这两种都可以：

```rust
dbg!(value); // 表达式的类型是 T
dbg!(&value); // 表达式的类型是 &T
```

宏展开时，会根据你传入的表达式直接生成代码，不需要提前把参数类型固定成 `Coin` 或 `&Coin`。

## `dbg!` 大致怎么工作

简化后可以理解成：

```rust
macro_rules! dbg {
    ($value:expr) => {{
        let temp = $value;
        eprintln!("{:?}", &temp);
        temp
    }};
}
```

注意它打印时使用：

```rust
&temp
```

也就是临时借用来打印；但是最终又把 `temp` 返回出去。

所以：

```rust
let x = String::from("hello");

let x = dbg!(x);
```

发生的是：

1. `x` 被移动进 `dbg!`
2. `dbg!` 临时借用它进行打印
3. `dbg!` 再把这个值返回
4. 新的 `x` 接住返回值

因此常写成：

```rust
let x = dbg!(x);
```

如果这样写：

```rust
let x = String::from("hello");

dbg!(x);
println!("{x}");
```

通常会报错，因为 `x` 被传入宏并移动了，而返回值又没有接住。

## 传引用则不会移动原值

```rust
let x = String::from("hello");

dbg!(&x);
println!("{x}");
```

这里传进去的表达式是：

```rust
&x
```

类型是：

```rust
&String
```

移动的是这个引用，而不是原来的 `String`，所以之后还能使用 `x`。

## 普通泛型函数也能同时接收值和引用

虽然普通函数不能把 `Coin` 参数同时当成 `&Coin`，但泛型函数可以让两次调用推断出不同的 `T`：

```rust
use std::fmt::Debug;

fn show<T: Debug>(value: T) -> T {
    println!("{value:?}");
    value
}
```

然后：

```rust
let x = String::from("hello");

show(&x); // T = &String
let x = show(x); // T = String
```

并不是一次调用中 `T` 同时代表值和引用，而是：

```text
show(&x)  → T 被推断为 &String
show(x)   → T 被推断为 String
```

所以 `dbg!` 看起来既能传值又能传借用，主要有两个原因：

> 它是宏，可以接受任意表达式；而且值和引用本来就是两种不同的表达式类型。

最重要的一点是：

```rust
dbg!(x)   // 可能移动 x
dbg!(&x)  // 只借用 x
```

# 我这个能都接收吗？

你这个函数**只能接收借用 `&Coin`**，不能同时接收 `Coin` 值：

```rust
fn value_in_cents(coin: &Coin) -> u8
```

可以这样调用：

```rust
let coin = Coin::Penny;

value_in_cents(&coin);
value_in_cents(&Coin::Penny);
```

不能直接这样：

```rust
value_in_cents(coin); // Coin，不是 &Coin
value_in_cents(Coin::Penny); // Coin，不是 &Coin
```

## 为什么 `match` 里不用写 `&Coin::Penny`

虽然 `coin` 的类型是 `&Coin`，你却可以写：

```rust
match coin {
    Coin::Penny => 1,
    Coin::Nickel => 5,
    Coin::Dime => 10,
    Coin::Quarter => 25,
}
```

这是 Rust 的 **match ergonomics**：模式匹配时会自动处理引用和解引用。概念上接近：

```rust
match *coin {
    Coin::Penny => 1,
    // ...
}
```

所以函数参数仍然是借用，只是 `match` 帮你省略了解引用。

## 这种简单枚举更适合按值传递

如果 `Coin` 的分支不携带 `String` 等复杂数据，可以让它实现 `Copy`：

```rust
#[derive(Clone, Copy)]
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => {
            println!("1 cent");
            1
        }
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

这样直接传：

```rust
let coin = Coin::Penny;

value_in_cents(coin);
value_in_cents(coin); // 可以，因为复制了一份
```

对于这种没有附加数据的小枚举，**`Copy` + 按值传递通常最自然**。你当前的 `&Coin` 写法也完全正确，只是调用时必须加 `&`。

# 什么叫绑定值的模式

“绑定值的模式”就是：

> **模式匹配成功时，把匹配到的数据取出来，赋给一个新变量。**

例如：

```rust
let x: Option<i32> = Some(5);

match x {
    Some(value) => println!("{value}"),
    None => println!("没有值"),
};
```

这里的：

```rust
Some(value)
```

是一个模式。其中 `value` 不是提前存在的变量，也不是在判断里面的值是否等于某个 `value`。

它的意思是：

1. 检查 `x` 是否是 `Some(...)`；
2. 如果是，就把 `Some` 里面的数据取出来；
3. 将它绑定到新变量 `value`。

所以当：

```rust
x = Some(5)
```

匹配：

```rust
Some(value)
```

之后，就相当于：

```rust
value = 5
```

## “只匹配”和“绑定值”的区别

只匹配分支，不取数据：

```rust
Some(_) => println!("有值")
```

这里 `_` 表示忽略内部数据。

匹配并绑定内部数据：

```rust
Some(value) => println!("值是 {value}")
```

这里 `value` 会接住内部数据。

可以对比：

```rust
match Some(5) {
    Some(_) => println!("有值，但我不关心是多少"),
    None => {}
}
```

以及：

```rust
match Some(5) {
    Some(number) => println!("值是 {number}"),
    None => {}
}
```

## 枚举分支携带多个值时

```rust
enum Message {
    Move(i32, i32),
    Quit,
}
```

模式可以同时绑定两个值：

```rust
let message = Message::Move(10, 20);

match message {
    Message::Move(x, y) => {
        println!("x = {x}, y = {y}");
    }
    Message::Quit => {}
}
```

这里：

```rust
Message::Move(x, y)
```

会把：

```rust
Message::Move(10, 20)
```

拆开并绑定为：

```text
x = 10
y = 20
```

## 结构体模式也可以绑定

```rust
struct User {
    name: String,
    age: u8,
}

let user = User {
    name: String::from("Tom"),
    age: 18,
};

match user {
    User { name, age } => {
        println!("{name}, {age}");
    }
}
```

这里 `name` 和 `age` 都是匹配时创建的新变量。

## 最容易混淆的地方

单独写一个变量名，本身也是模式：

```rust
match 5 {
    number => println!("{number}"),
}
```

`number` 会匹配任何值，并把整个值绑定给自己。

因此可以记成：

```text
5              字面量模式：只匹配数字 5
_              通配符模式：匹配但忽略
number         绑定模式：匹配任何值并绑定给 number
Some(number)   匹配 Some，并绑定里面的值
```

所以“绑定值的模式”本质上就是：

> **一边检查数据的形状，一边把其中需要的部分取出来命名。**
